Core Features

This section details the primary functionalities of the tool. Interact with each card to reveal the specific requirements for each feature, from the layout design to the error handling protocol.

Split-Screen Layout +

A two-pane interface is required: a code editor on the left and a live preview on the right.

Live Transpilation & Rendering +

The tool must automatically convert JSX into browser-compatible code and update the preview in near real-time. A 500ms debounce mechanism should be used to optimize performance.

Robust Error Handling +

The application must not crash on code errors. Instead, it should display a clear error message in the preview pane and clear any previous successful render.

Default Code Example +

The editor should be pre-populated with a simple but functional React component to serve as a clear example for users. This component should demonstrate state and event handling.

Technology Stack

This section outlines the specified technologies for building the application. Each technology was chosen to meet the project's requirements for modern development, performance, and maintainability.

⚛️ React 18+

Functional components and hooks (`useState`, `useEffect`, `useRef`) are required. The React 18 UMD build must be loaded dynamically to isolate it from the host environment.

💨 Tailwind CSS

All styling must be implemented using Tailwind's utility classes to ensure a fully responsive and modern design.

📦 Babel Standalone

This client-side transpiler must be loaded dynamically to convert user-written JSX code into standard JavaScript directly in the browser.

Implementation Roadmap

This section outlines the recommended development steps. Following this sequence ensures a structured approach, from setting up the basic components to implementing the core rendering logic.

1. Component Structure

Create the main components: `App`, `CodeEditor`, and `Preview`. Set up the basic state management for code and errors using `useState`.

2. Dynamic Script Loading

Implement a `useEffect` hook to dynamically load the Babel and ReactDOM v18 scripts. Use a state flag to track when they are ready.

3. Rendering Logic

Create the core `useEffect` hook that depends on code changes. Inside, transpile the code with Babel and use the loaded ReactDOM's `createRoot()` API to render the component.

4. Error Handling & Polish

Wrap the rendering logic in a try/catch block. On error, update the error state and unmount the React root. Finalize styling and responsiveness.
